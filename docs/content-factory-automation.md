# Как работает контент-завод

## Что уже автоматизировано

1. Telegram-посты лежат в `docs/channel-posts/`.
2. Расписание Telegram-постов лежит в `config/channel_schedule.json`.
3. GitHub Actions workflow `Channel scheduler` каждый час проверяет расписание.
4. Если пост созрел и его нет в `data/channel_published.json`, workflow отправляет его в канал.
5. После отправки workflow записывает `message_id`, чтобы не было дублей.
6. Админка показывает вкладку "Календарь постов" с предпросмотром постов.

Посты канала публикуются с чистыми горизонтальными карточками из `assets/channel-posts/`. Сторис-картинки из `assets/stories/` в посты не ставим, чтобы не тянуть в ленту полоски и лишний story-интерфейс.

## Как смотреть видео и картинки

Визуальная витрина креативов генерируется командой:

```bash
python scripts/generate_social_creative_preview.py
```

После генерации открывай:

```text
docs/social-creative-preview.html
```

Там видно, как выглядят вертикальные TikTok/Reels/Shorts mockup-ролики: хук, кадры, подпись, CTA и ссылка в бота.

## Что пока не автоматизировано

Автопубликация в TikTok, Instagram Reels и YouTube Shorts пока не подключена. Эти платформы требуют отдельные бизнес-аккаунты, доступы к API, правила публикации и иногда ручное подтверждение приложения.

Сейчас автоматизирована подготовка: сценарии, подписи, deep links, предпросмотр и проверка безопасного текста. Публикация TikTok/Reels пока остается ручной или через отдельный approved-планировщик.

## Как будет работать полный цикл

1. SMM добавляет идею в `config/content_campaigns.json`.
2. Скрипт строит deep link с `source/campaign/content`.
3. Скрипт генерирует HTML-предпросмотр креативов.
4. Дизайнер/видео-продюсер собирает MP4 по кадрам из предпросмотра.
5. Оператор публикует ролик в TikTok/Reels/Shorts.
6. Метрики заносятся в таблицу: просмотры, клики, старты бота, paywall, оплаты.
7. Лучшие хуки попадают в следующий цикл.

## Следующий уровень автоматизации

Чтобы сделать полностью автоматический завод, нужны:

* папка `social/video/reels-export/` для готовых MP4;
* manifest-файл с датой публикации, платформой, подписью и файлом видео;
* подключение TikTok Content Posting API;
* подключение Meta Graph API для Instagram Reels;
* отдельный workflow, который берет approved MP4 и публикует по расписанию;
* безопасный режим, где первый запуск делает только dry-run.
