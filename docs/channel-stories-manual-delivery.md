# Сторис канала: генерация и доставка админу

Telegram Bot API не даёт боту публиковать stories от имени канала `@trynit_ai`.

Рабочий контур для канала:

1. Workflow `Story scheduler` каждый день генерирует story-картинки.
2. Бот отправляет готовую картинку и caption админу в личку.
3. Админ вручную публикует сторис от аккаунта, у которого есть право публиковать stories канала.
4. Workflow записывает доставку в `data/story_delivered.json`, чтобы не присылать дубль.

## Где лежит расписание

```text
config/story_schedule.json
```

В каждом элементе есть:

- `publish_at` — когда отправить админу;
- `background_file` — фон;
- `image_file` — куда сохранить картинку;
- `bubble_text` — текст на самой картинке;
- `caption` — текст/ссылка для ручной публикации.

## Как проверить локально

```powershell
python scripts\generate_story_assets.py
python scripts\deliver_scheduled_stories.py --dry-run --now 2026-05-20T11:00:00+03:00
```

## Как включить доставку в GitHub

Добавить secret:

```text
STORY_DELIVERY_CHAT_ID=<твой Telegram user_id>
```

Если secret не задан, локально скрипт может использовать `OWNER_ID` из `.env`.

## Как запустить вручную

```text
Actions → Story scheduler → Run workflow → dry_run=false → limit=1
```

После этого бот пришлёт админу:

- PNG-картинку сторис;
- текст на картинке;
- caption и ссылку для публикации;
- ID сторис из расписания.

## Что делать админу

1. Скачать/открыть картинку из сообщения бота.
2. Открыть канал `@trynit_ai`.
3. Создать story канала вручную.
4. Вставить caption/ссылку из сообщения бота.
5. Опубликовать.

Это безопаснее, чем userbot: не нужны пользовательские сессии, нет риска блокировки аккаунта за неофициальную автоматизацию, и весь контент всё равно готовится автоматически.
