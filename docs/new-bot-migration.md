# Перенос Нити на новый Telegram-бот

Цель: запустить нового бота в едином стиле Нити без потери управляемости, платежей и проверок.

Готовые тексты для BotFather, username-варианты, команды, первый пост и сообщение-мост лежат в `docs/new-bot-profile.md`.

## Что нужно подготовить

- Новый бот в BotFather: имя, username, описание, about, команды.
- Новый `BOT_TOKEN` для окружения продакшена.
- Аватар бота: `assets/nit-avatar.png`.
- Стартовая карточка: `assets/launch-card.png`.
- Ссылка для продвижения: `https://t.me/asknitai_bot`.
- Если используются платежи Telegram, проверить provider token и соответствие нового бота платежному провайдеру.

## Единый стиль

- Образ: темный премиальный фон, световая нить, спокойный человеческий AI-компаньон.
- Главный визуал запуска: `assets/launch-card.png`.
- Аватар персонажа/бота: `assets/nit-avatar.png`.
- Тон текста: не “ассистент”, а живой собеседник, который держит контекст и помогает продолжить мысль.
- Основной CTA: “Напиши как есть. Нить подхватит.”

## Шаги переноса

1. Создать нового бота в BotFather и получить новый `BOT_TOKEN`.
2. Поставить аватар `assets/nit-avatar.png` в профиль нового бота.
3. Обновить секрет `BOT_TOKEN` в GitHub Actions или на сервере.
4. Если меняется username, обновить `launch.bot_username`, UTM-ссылки и контент-план в админке.
5. Запустить `python scripts/prelaunch_check.py`.
6. Выполнить деплой через `deploy/run_github_deploy.ps1`.
7. Проверить `/start`: должна прийти стартовая карточка `assets/launch-card.png`, приветствие и CTA.
8. Проверить оплату тестовым тарифом или sandbox-сценарием, если он доступен.
9. Проверить инициативные сообщения и вкладку админки с логами инициативы.

## Что не переносится автоматически

- Подписчики старого Telegram-бота не переезжают в нового автоматически.
- История чатов пользователей в Telegram не переносится между ботами.
- Если база остается той же, внутренние профили пользователей сохранятся, но Telegram ID должны прийти через нового бота заново.

## Рекомендованный запуск

Сначала держать старого бота как мост: отправить пользователям короткое сообщение с причиной переезда и ссылкой на нового бота. Новый бот должен сразу выглядеть цельно: аватар, стартовая карточка, описание, меню, приветствие, paywall и канал в одном визуальном языке.
